# 多模型协同与金融宏观数据源整合规范 (Multi-Model RAG & Data Sources Integration Spec)

## Why
现有系统虽然搭建了多智能体与多模型的初步框架，但针对垂直领域（锂电池企业分析与投资）的权威数据获取需要进一步增强，以确保 RAG 的知识检索绝对精准。同时，为了提高系统的竞争力和分析能力，必须深度整合三大交易所的财务报告、东方财富的行业与个股研报，以及国家统计局的宏观经济数据。为了高效处理这些复杂且多样化的任务，需要进一步优化 deepseek-reasoner、GLM-5、Qwen3.5-Plus 三款大模型的协同工作机制，使其在不同智能体中发挥最大效能，确保系统能够做出精准的决策，同时保持“仅填API密钥即可运行”的极简易用性。

## What Changes
- **新增定向数据获取智能体（Data Agent）**：针对特定金融与宏观网站（上交所、深交所、北交所、东方财富、国家统计局）进行定时或按需的数据抓取与解析。
- **建立国家统计局数据的认证访问机制**：支持账号凭证或 Cookie 模拟登录配置。
- **优化多模型协同编排引擎**：根据任务特性自动路由至最合适的大模型，明确职责边界：
  - **deepseek-reasoner**：专攻复杂逻辑推理、数学模型指标的深度解读、研报数据的交叉验证与核心投资决策推导。
  - **GLM-5**：专攻超长文本阅读与理解，负责从数十页的财务报告、长篇研报中抽取关键信息、总结与结构化处理。
  - **Qwen3.5-Plus**：作为中枢大脑与交互窗口，负责日常对话响应、用户意图快速理解、基础信息检索、系统指令生成与工作流编排。
- **升级 RAG 知识库系统**：建立针对金融公告、研报、宏观数据的定制化文本切片、结构化存储与混合向量检索机制，保证信息获取的高效准确。
- **完善配置层**：确保系统在接入更多数据源和模型后，依然保持“开箱即用”，仅需配置相关的 API 密钥（与必要的数据源账号凭证）即可顺利运行。

## Impact
- Affected specs: 外部数据源采集规范、RAG知识库构建规范、多模型编排策略、Agent工作流与协作机制。
- Affected code: 数据抓取与解析模块（新增）、向量检索模块、模型路由与调度中心、统一环境配置模块（`.env`管理）。

## ADDED Requirements
### Requirement: 权威金融与宏观数据接入
系统 SHALL 能够定时或按需从指定的交易所、研报平台及国家统计局获取最新数据，并作为分析的强有力上下文补充。

#### Scenario: 抓取最新财报与公告
- **WHEN** 触发企业财报更新任务或用户请求特定公司数据时
- **THEN** 智能体从上交所、深交所或北交所检索该公司的最新定期报告，并利用 GLM-5 提取核心财务指标。

#### Scenario: 抓取行业与个股研报
- **WHEN** 用户请求锂电池行业分析或某只股票的投资建议时
- **THEN** 智能体从东方财富网检索相关研报，提取分析师观点与预测数据进行结构化存储。

#### Scenario: 获取宏观经济数据
- **WHEN** 投资或运营分析需要宏观环境因子（如 PPI、CPI、工业增加值）时
- **THEN** 智能体利用预先配置好的凭证从国家统计局获取对应周期的宏观数据。

### Requirement: 三大模型优势互补的智能体协同编排
系统 SHALL 建立一个智能路由机制，根据任务的上下文长度、逻辑复杂度和响应时间要求，将任务分配给最合适的大模型，并支持结果的交叉审校与融合。

#### Scenario: 处理复杂投资决策与诊断推导
- **WHEN** 决策智能体接收到需要深度逻辑推理的任务（例如：基于毛利承压模型和经营质量变化动态评价模型的结果，给出投资建议）
- **THEN** 任务调度器将任务分配给 deepseek-reasoner 进行思维链（CoT）深度推导，确保决策的高质量。

#### Scenario: 处理超长财报与研报文本
- **WHEN** 信息抽取智能体需要从几十页的年报或行业研报中提取特定数据
- **THEN** 任务调度器将长文本全量交由 GLM-5 进行阅读、摘要与关键信息精准抽取。

#### Scenario: 用户日常交互与快速意图理解
- **WHEN** 用户在前端发起对话或发出系统指令
- **THEN** 交互智能体使用 Qwen3.5-Plus 进行快速意图识别，并将子任务分发给其他智能体，最后由其汇总生成流畅的回复。

## MODIFIED Requirements
### Requirement: 极简配置启动
系统 SHALL 在引入复杂爬虫架构与三模型协同后，依然保持极简配置原则。
**修改点**：除三大模型的 API 密钥外，允许在配置文件中追加可选的国家统计局等数据源的访问凭证（账号/密码或 Cookie），系统在缺失这些可选凭证时能够平滑降级（如仅使用公开免登录数据或给出明确提示），而不影响系统的整体运行。

## REMOVED Requirements
无。
