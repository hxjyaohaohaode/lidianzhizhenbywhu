# 涟漪式放射记忆树与缩放拖拽交互 Spec

## Why
当前系统“记忆中的你”界面节点采用了简单的螺旋布局，且视图仅支持原生的基础拖拽（通过原生 scroll 控制）。用户希望以“中心用户数据为圆心，一级记忆点绕中心排成一圈，点击后如涟漪扩散般向外层展开二级节点”的同心圆（Concentric / Radial）构图方式，这种方式更能凸显“记忆源自中心向外发散”的语义。同时，为了在有限的屏幕空间内更好地浏览多层展开后的庞大图谱，系统需要提供画布级别的缩放（Zoom）和自由平移（Pan）能力。

## What Changes
- 重构坐标计算算法，将当前的螺旋（Spiral）布局改为同心圆放射形（Radial/Concentric）布局：
  - 中心点（Level 0）置于画布正中央。
  - 一级节点（Level 1）均等分布在半径为 R1 的第一个圆环上。
  - 展开一级节点时，其下属的二级节点（Level 2）以一级节点为圆心，沿特定角度范围（扇形）或半径为 R2 的外圈排布，形成“涟漪扩散”的视觉效果。
- 引入或实现画布的平移与缩放（Pan & Zoom）机制：
  - 通过鼠标滚轮（Wheel）实现视图的放大与缩小（控制 `transform: scale()`）。
  - 通过鼠标拖拽背景（MouseDown + MouseMove）实现视图的自由平移（控制 `transform: translate()`）。
  - 确保连线（SVG lines）与节点（Divs）在同一个缩放层级中同步变化。
- 为节点的展开/收起添加过渡动效（Transition），使位置变动和节点出现/消失更具涟漪扩散的平滑感。

## Impact
- Affected specs: 记忆页节点坐标布局算法、画布交互事件（缩放与拖拽）
- Affected code: `src/web/App.tsx`, `src/web/styles.css`

## ADDED Requirements
### Requirement: 涟漪扩散式的放射布局
系统 SHALL 将一级节点均匀排列在围绕中心节点的第一个圆环上。当点击某一级节点展开时，其二级节点必须在更外层的同心圆或扇形区域上放射状生成。

#### Scenario: 节点展开的涟漪效果
- **WHEN** 用户点击一个处于闭合状态的一级节点
- **THEN** 系统计算出外层圆环的对应坐标，使新增的二级节点带有平滑的位移动画出现在该区域，宛如水波纹（涟漪）向外扩散。

### Requirement: 画布的缩放与自由拖拽
系统 SHALL 支持对整个记忆节点图谱进行无级缩放和平移，以便用户在图谱展开变大时能够看清全局或聚焦局部细节。

#### Scenario: 使用鼠标缩放和平移图谱
- **WHEN** 用户在背景空白处使用鼠标滚轮滚动
- **THEN** 整个记忆图谱连同中心、节点与连线一起放大或缩小。
- **WHEN** 用户在背景空白处按下鼠标左键并移动
- **THEN** 图谱跟随鼠标进行二维平移，释放鼠标后停止平移。

## MODIFIED Requirements
### Requirement: 视图滚动容器替换
原先依赖容器的 `scrollLeft/scrollTop` 来实现基础平移的方式，将被统一替换为控制图谱容器的 `transform: translate(x, y) scale(z)` 样式，从而实现高性能且无边界的 Pan & Zoom 体验。
