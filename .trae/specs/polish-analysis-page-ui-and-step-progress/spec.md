# 分析页面视觉升级与智能体进度条增强 Spec

## Why
当前分析页面的视觉设计需要按照线框图进行优化，确保界面美观且对话体验流畅。同时，现有的进度条仅显示百分比和文字描述，缺乏对智能体具体工作步骤的可视化展示，用户无法直观了解 AI 当前执行到了哪一步。

## What Changes
- 重构分析工作台头部区域（tag 导航栏、模式切换、历史/新建/删除对话功能）的视觉层次与布局
- 优化消息气泡样式，提升对话阅读体验与视觉美感
- 重新设计进度条组件：从简单的百分比条升级为**分步骤进度指示器**，清晰展示智能体当前执行阶段
- 优化右侧边栏面板的视觉呈现
- 优化底部审计数据看板的图表区域布局
- 确保深色/浅色主题下均有一致的精美视觉效果

## Impact
- Affected specs: 无直接依赖的已完成 spec
- Affected code: `src/web/styles.css`（主要）、`src/web/App.tsx`（InvAna 组件中的进度与布局部分）

## ADDED Requirements

### Requirement: 分步骤智能体进度指示器

系统 SHALL 在分析工作台中提供一个**分步骤进度指示器**，当智能体正在工作时实时显示当前执行到的具体步骤。

#### 场景 1: 智能体运行中显示分步进度
- **WHEN** 用户发送问题后智能体开始处理（`loading === true`）
- **THEN** 系统在消息流底部显示一个分步骤进度指示器，包含：
  - 当前阶段名称（如"数据采集"、"行业分析"、"证据整理"、"报告生成"等）
  - 总步骤数与当前步骤索引（如 "步骤 2/5"）
  - 一个带步骤节点的可视化进度条，每个节点代表一个步骤，已完成的步骤高亮显示，当前步骤带有脉冲动画
  - 当前步骤的简要描述文字

#### 场景 2: 智能体空闲时隐藏详细进度
- **WHEN** 智能体未处于运行状态（`loading === false`）
- **THEN** 分步骤进度指示器自动隐藏或折叠为简洁状态；主进度卡片仍可在有历史进度时以摘要形式展示

#### 场景 3: 不同分析模式的步骤定义
- **WHEN** 用户切换到「行业状况」模式
- **THEN** 进度步骤定义为：数据采集 → 行业扫描 → 趋势识别 → 风险评估 → 报告输出
- **WHEN** 用户切换到「深度解析」模式
- **THEN** 进度步骤定义为：问题澄清 → 条件收集 → 数据研究 → 深度分析 → 结论输出
- **WHEN** 用户切换到「正式辩论」模式
- **THEN** 进度步骤定义为：论点构建 → 第一轮辩论 → 第二轮辩论 → 第三轮辩论 → 裁判总结

### Requirement: 分析工作台视觉层次优化

系统 SHALL 对分析工作台进行全面视觉优化，使其符合现代设计美学标准。

#### 场景 4: 头部区域精致化
- **WHEN** 用户进入分析工作台页面
- **THEN** 头部区域包含：
  - tag 导航栏（模式切换标签），使用胶囊式选中态设计
  - 右侧操作按钮组（历史对话、新建对话、删除当前对话），图标+文字排列整齐
  - 会话元信息栏（附件数、更新时间等），以轻量标签形式展示

#### 场景 5: 对话区域美化
- **WHEN** 消息在对话区域中渲染
- **THEN** 
  - 用户消息气泡使用渐变色（蓝紫渐变），圆角气泡带阴影
  - AI 消息气泡使用毛玻璃效果背景，边框微发光
  - 辩论消息带角色标识徽章和轮次标签
  - 系统消息使用警告色系背景区分
  - 打字动画（三点跳动）使用品牌色渐变

#### 场景 6: 输入区域精致化
- **WHEN** 用户聚焦输入框
- **THEN** 输入区域呈现聚焦态：边框变为品牌色，外圈出现柔和光晕，上传按钮和发送按钮有微交互反馈

## MODIFIED Requirements

### Requirement: 主进度卡片增强

现有 `.iwb-progress-card` 组件 SHALL 从纯百分比展示升级为包含步骤概览的综合进度卡片。

- **修改内容**: 在保留原有百分比进度条的基础上，增加步骤节点指示器（小圆点 + 连接线），每个步骤节点可显示步骤名称 tooltip
- **交互**: 点击步骤节点可展开查看该步骤的详细信息（如耗时、产出物摘要）

## REMOVED Requirements
无
