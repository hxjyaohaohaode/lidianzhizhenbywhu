# 记忆节点排布与玻璃质感连线优化 Spec

## Why
目前记忆节点采用了基础的放射形排布与简单的 SVG 实线连线。但在多级节点展开时，节点间的连线过于生硬且单调，缺乏系统整体设定的“磨砂玻璃与高质感”氛围。为了让节点图谱看起来更有科技感、空间感和连结感，需要对节点间的 SVG 连线应用“玻璃质感/发光/半透明渐变”效果，并进一步优化排布角度，确保不同层级节点展开时视觉张力更饱满、连线不生硬。

## What Changes
- 优化 SVG 连线的样式与渲染方式。不再使用单纯的单色 `stroke`，而是通过引入 SVG `<defs>`（如 `<linearGradient>` 和 `<filter>`）为连线赋予发光、半透明渐变以及柔和的高斯模糊阴影，使其呈现“发光的玻璃光纤”质感。
- 微调 Level 1 与 Level 2 的 `generateRadialPos` 坐标计算参数，可能的话引入稍微带有曲线感（或错落感）的排布，避免绝对直线导致的呆板。
- 保证连线的起始点与结束点精确对齐到节点的边缘或中心（考虑节点的尺寸 offset）。
- 确保玻璃质感连线在深色与浅色模式下均有良好的对比度表现（例如深色下呈现明亮的蓝紫光晕，浅色下呈现通透的冰蓝/紫晕）。

## Impact
- Affected specs: 记忆树图谱的连线样式与排布微调
- Affected code: `src/web/styles.css`, `src/web/App.tsx`

## ADDED Requirements
### Requirement: 玻璃发光质感的连线
系统 SHALL 将多级记忆节点之间的连线升级为具有发光渐变与半透明质感的样式。连线看起来应该像连接玻璃卡片的光源或毛玻璃流光。

#### Scenario: 节点展开时的连线呈现
- **WHEN** 记忆图谱初始化或用户点击展开二级节点时
- **THEN** 新增的 SVG `<line>`（或 `<path>`）会带有 `filter` 发光效果与半透明颜色，使得图谱层级关系看起来更具现代感与科技感。

## MODIFIED Requirements
### Requirement: 放射状排布的视觉张力
系统在进行放射状计算时，应考虑增加不同节点之间的角度错落，或让二级节点的扇形夹角与父节点产生更好的包裹感，避免节点过度拥挤。
