# 图表稳定性与身份选择改进规格

## Why
用户反馈图表需要更稳定、更大、布局更美观，同时希望每次进入页面时都能选择身份。当前图表可能在某些情况下存在性能问题，尺寸不够突出，布局有待优化。身份选择流程仅在初次加载时出现，用户无法在应用内快速切换身份。

## What Changes
- **图表稳定性优化**：检查并修复图表渲染中的潜在崩溃点，优化性能，确保图表在不同数据量下稳定运行。
- **图表尺寸增大**：调整图表容器比例，减少边距，使图表区域更大，提升数据可视化效果。
- **布局美观优化**：优化图表内部元素的间距、对齐、颜色搭配，确保整体布局协调美观。
- **身份选择流程改进**：在应用内添加显式的身份切换入口，允许用户随时切换企业端/投资端身份，同时确保每次页面刷新或重新访问时都能进行身份选择（可配置）。

## Impact
- **受影响的规格**：enhance-charts（图表升级）、redesign-ui（用户界面设计）
- **受影响的代码**：chart-system.tsx（图表系统）、RoleScreen.tsx（身份选择组件）、AppContext.tsx（应用状态管理）、相关样式文件

## ADDED Requirements
### 要求：图表稳定性检查与优化
系统应确保所有图表组件在数据加载、交互、缩放等操作下保持稳定，无崩溃或性能卡顿。

#### 场景：大数据量渲染
- **当** 图表需要渲染大量数据点（如超过1000个点）时
- **那么** 图表应保持流畅渲染，无明显卡顿，且不导致浏览器崩溃

#### 场景：频繁交互
- **当** 用户快速进行hover、点击、缩放等交互操作时
- **那么** 图表应响应及时，无延迟或闪烁

### 要求：图表尺寸增大
系统应增大图表显示区域，使图表更突出，数据更易读。

#### 场景：查看图表详情
- **当** 用户查看任何图表时
- **那么** 图表区域应占容器面积的90%以上（当前为85%+），边距和空白区域减少

### 要求：布局美观优化
系统应优化图表内部布局，包括标题、图例、坐标轴、数据标签等的间距、对齐和颜色搭配。

#### 场景：多图表并列
- **当** 多个图表在同一看板中显示时
- **那么** 图表之间应有协调的间距和对齐，整体视觉效果统一美观

### 要求：身份选择随时可用
系统应提供显式的身份切换入口，允许用户在应用内随时切换身份。

#### 场景：切换身份
- **当** 用户在企业端界面希望切换到投资端时
- **那么** 用户可通过页面顶部的身份切换器快速切换，无需重新加载页面或返回初始角色选择界面

#### 场景：重新访问页面
- **当** 用户刷新页面或重新访问应用时
- **那么** 系统应显示身份选择界面（可配置为每次都显示或仅当本地无记录时显示），确保用户能选择身份

## MODIFIED Requirements
### 要求：现有身份选择流程
修改现有身份选择流程，使其不仅限于初次加载，而是每次页面刷新时都显示角色选择界面（除非用户明确选择记住身份）。

**修改内容**：调整AppContext中的状态逻辑，使`appState`在每次页面加载时重置为'role'（除非本地存储中有“记住身份”标志）。同时，在应用界面添加身份切换按钮。
