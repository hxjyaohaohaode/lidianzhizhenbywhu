# 面向双人群的智能诊断平台规范

## Why
当前项目只有业务设计方案与数学模型文档，缺少可落地的底层软件架构、智能体协作机制、模型调用逻辑与高质量交付规范。需要先定义一套可实施的全栈规范，确保系统以两个数学模型为核心、只需填写大模型 API 密钥即可运行，并具备“企业运营分析”与“投资人员”双端高质感体验。

## What Changes
- 定义双入口 Web 应用的总体架构，包括首屏身份选择、双角色信息采集、首页、分析、设置与“记忆中的你”子系统
- 定义以“毛利承压模型”和“经营质量变化动态评价模型”为核心的分析引擎边界、输入输出与计算流水线
- 定义多模型协作层，支持 deepseek-reasoner、GLM-5、Qwen3.5-Plus 的统一配置、路由、降级与审校
- 定义 Agent 协作系统，覆盖数据采集、指标解释、研判生成、投资建议、行业洞察、任务编排与记忆管理
- 定义 RAG 与即时信息获取链路，包括网页检索、可信度交叉验证、知识缓存、引用回传与安全边界
- 定义仅配置 API 密钥即可启动的软件配置模式，包括环境变量、默认工作流、模型适配器与可替换供应商层
- 定义高质感前端体验要求，包括动画缓冲页、动态仪表盘、对话分析页、3D 玻璃态“记忆中的你”界面
- 定义可测试、可观测、可扩展的工程基线，包括日志、埋点、任务状态、异常回退与质量门禁

## Impact
- Affected specs: 双角色交互流程、数学分析引擎、多模型编排、Agent 工作流、RAG 检索、记忆管理、系统配置、安全合规、前端体验
- Affected code: Web 前端应用、服务端 API 层、模型编排服务、数学计算模块、数据采集与检索模块、向量检索模块、会话与记忆模块、配置中心、可观测性模块、测试体系

## ADDED Requirements
### Requirement: 双角色统一入口
系统 SHALL 提供一个具有高质感动画缓冲效果的统一入口，并在首次使用时引导用户选择“企业运营分析”或“投资人员”身份。

#### Scenario: 首次进入系统
- **WHEN** 用户首次打开系统
- **THEN** 系统显示动画缓冲界面并进入身份选择流程
- **THEN** 系统保存身份选择结果用于后续默认导航与工作流装配

#### Scenario: 再次进入系统
- **WHEN** 用户已完成身份选择并再次打开系统
- **THEN** 系统根据已保存身份直接恢复到对应的角色工作台

### Requirement: 企业运营分析数据采集流
系统 SHALL 为企业运营分析用户提供结构化的数据采集流程，并根据“是否具备最近 4 个季度历史数据”动态调整表单范围。

#### Scenario: 企业用户拥有 4 个季度历史数据
- **WHEN** 企业用户选择“有最近 4 个季度历史运营数据”
- **THEN** 系统要求完整采集当前季度及最近 4 个季度的关键经营数据
- **THEN** 系统在采集页底部持续显示保密承诺文案

#### Scenario: 企业用户缺少完整历史数据
- **WHEN** 企业用户选择“没有最近 4 个季度历史运营数据”
- **THEN** 系统仅要求采集当前季度与四个季度前的必要对比数据
- **THEN** 系统仍可完成模型计算并标注分析置信度限制

### Requirement: 投资人员画像与偏好采集流
系统 SHALL 为投资人员提供投资现状、已投企业、资金成本与投资兴趣等信息采集流程，并基于该画像驱动后续智能体工作模式。

#### Scenario: 投资者完成画像初始化
- **WHEN** 投资人员首次完成基础信息填写
- **THEN** 系统生成投资偏好档案并用于推荐、对话与趋势提醒

### Requirement: 以数学模型为核心的分析引擎
系统 SHALL 将“毛利承压模型”和“经营质量变化动态评价模型”作为所有企业分析与投资分析结果的核心计算基础，大模型仅负责解释、扩展与交互优化。

#### Scenario: 生成企业诊断结果
- **WHEN** 用户提交企业经营数据
- **THEN** 系统先运行两个数学模型得到结构化指标、评分、趋势与风险信号
- **THEN** 大模型只能读取数学模型结果与上下文材料进行解释、建议与问答扩写

#### Scenario: 生成投资分析结果
- **WHEN** 投资人员查看企业分析或请求推荐
- **THEN** 系统将企业数学模型输出、行业信息与用户画像联合输入决策工作流
- **THEN** 推荐结论必须可追溯到数学指标、证据片段与模型推理链摘要

### Requirement: 多模型协作与统一适配
系统 SHALL 提供统一的大模型适配层，使系统仅需配置各模型 API 密钥即可运行，并支持任务级路由、失败降级与多模型交叉审校。

#### Scenario: 用户完成 API 配置
- **WHEN** 管理员或部署者填写 deepseek-reasoner、GLM-5、Qwen3.5-Plus 的 API 密钥
- **THEN** 系统自动完成模型适配器初始化与健康检查
- **THEN** 无需修改业务代码即可启用全量智能体工作流

#### Scenario: 某个模型不可用
- **WHEN** 某个外部模型调用失败或超时
- **THEN** 系统自动切换到预设备援模型或精简工作流
- **THEN** 用户界面展示已降级但仍可用的结果状态

### Requirement: Agent 协作工作流
系统 SHALL 建立可编排的多 Agent 体系，至少包含数据理解 Agent、数学分析 Agent、行业检索 Agent、证据审校 Agent、表达生成 Agent、任务编排 Agent 与记忆管理 Agent。

#### Scenario: 企业用户发起分析对话
- **WHEN** 企业用户在分析页提出问题
- **THEN** 任务编排 Agent 根据问题类型调用相关 Agent 并并行收集结果
- **THEN** 数学分析 Agent 优先输出基于模型的结构化判断
- **THEN** 证据审校 Agent 对网页检索结果和模型生成结果进行一致性检查
- **THEN** 表达生成 Agent 生成面向用户的最终回复与可视化建议

#### Scenario: 投资人员切换分析模式
- **WHEN** 投资人员选择“行业状况分析”“投资推荐”或“深度解析”
- **THEN** 系统切换对应的 Agent 工作流模板与提示策略
- **THEN** 保持统一会话上下文与可追溯证据输出

### Requirement: 即时信息 RAG 与可信度控制
系统 SHALL 支持基于网页数据的即时检索增强生成，并通过多源交叉验证与可信度评分控制回答质量。

#### Scenario: 需要外部行业信息
- **WHEN** 分析任务缺少实时行业、市场或企业公开信息
- **THEN** 系统启动检索流水线抓取网页数据并进行清洗、切片、索引与排序
- **THEN** 多模型审校流程对关键信息进行可信度比对
- **THEN** 最终回答返回引用来源摘要与可信度说明

### Requirement: 高质感分析工作台
系统 SHALL 为两类角色提供高质感首页、分析页与设置页，并保持视觉统一、交互顺滑与信息密度可控。

#### Scenario: 企业首页展示
- **WHEN** 企业用户进入首页
- **THEN** 系统展示毛利承压、经营质量、风险信号、趋势判断与关键经营仪表盘

#### Scenario: 投资首页展示
- **WHEN** 投资人员进入首页
- **THEN** 系统展示已投企业动态、趋势预测、机会提醒与风险提示

### Requirement: “记忆中的你”私有记忆子系统
系统 SHALL 提供一个带动画缓冲页的私有记忆管理界面，并以可滑动的树状 3D 玻璃结构展示已记录的用户记忆节点。

#### Scenario: 用户查看记忆节点
- **WHEN** 用户打开“记忆中的你”
- **THEN** 系统以中心节点为核心展示可滑动的树状结构
- **THEN** 每个节点默认显示关键摘要，点击后展示详细记忆内容
- **THEN** 背景、光效与节点反射效果随交互动态变化

### Requirement: 安全配置与隐私保护
系统 SHALL 通过环境配置管理所有外部模型密钥与敏感参数，不在界面、日志与前端代码中暴露密钥，并对企业上传数据执行隔离、脱敏与最小化使用原则。

#### Scenario: 处理企业敏感经营数据
- **WHEN** 系统接收企业经营数据
- **THEN** 系统只将经过策略许可的数据发送给必要的后端服务
- **THEN** 敏感字段不出现在客户端持久化与普通日志中

### Requirement: 可观测性与质量门禁
系统 SHALL 提供从前端交互到后端工作流的可观测性，并对模型调用、任务编排、数学计算与异常回退建立质量门禁。

#### Scenario: 工作流异常
- **WHEN** 任一 Agent、模型调用或数据计算出现异常
- **THEN** 系统记录结构化追踪信息并展示用户可理解的失败反馈
- **THEN** 系统尽可能提供回退结果或下一步建议

## MODIFIED Requirements
### Requirement: 智能体输出逻辑
系统 SHALL 从“单次问答式智能体”升级为“数学模型优先、工作流驱动、多 Agent 协作、证据可追溯”的复合智能系统，所有高价值结论必须经过结构化分析链路而非直接生成。

## REMOVED Requirements
### Requirement: 纯大模型直接生成核心结论
**Reason**: 直接依赖大模型生成核心经营或投资结论会削弱数学模型的主导地位，并降低结果可解释性与可信度。
**Migration**: 将所有核心结论先映射为数学指标、证据摘要与中间结构化结果，再交由表达生成层组织输出。
