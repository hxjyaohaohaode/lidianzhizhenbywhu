# 稳定界面布局并增强深色磨砂真实感 Spec

## Why
当前系统在复杂界面、弹层和多区域切换场景下，仍需进一步强化布局稳定性，避免出现错位、遮挡、尺寸抖动和显示异常。与此同时，深色模式虽然已有质感基础，但内容层的材质表达还不够真实，需要增加适度磨砂与微纹理层次，提升高端感与沉浸感。

## What Changes
- 梳理全局布局容器、卡片、弹层和滚动区域的尺寸约束、对齐规则与溢出策略，降低界面错位和显示错误风险。
- 为高频交互区域建立更稳定的弹层、面板和内容区样式基线，确保不同页面状态切换下视觉结构一致。
- 强化深色模式下的背景与面板材质表现，为内容层增加克制的磨砂、微弱噪点和高光边缘，避免脏污感和可读性下降。
- 补充与主题切换、弹层显示、关键页面结构相关的验证，确保稳定性优化不会破坏现有交互体验。

## Impact
- Affected specs: 全局界面稳定性、深色模式视觉主题、弹层与工作区布局一致性
- Affected code: `src/web/styles.css`, `src/web/App.tsx`, `src/web/App.test.tsx`

## ADDED Requirements
### Requirement: 稳定的界面布局基线
系统 SHALL 为主框架、卡片、弹层、滚动容器和分栏区域提供统一且稳定的布局约束，避免在窗口尺寸变化、内容增减和状态切换时出现错位、挤压、遮挡或异常溢出。

#### Scenario: 主界面稳定显示
- **WHEN** 用户在不同页面间切换，或打开弹层、历史记录、详情面板等内容
- **THEN** 各区域的定位、对齐、层级和滚动行为保持稳定，不出现明显错位、裁切或重叠错误

### Requirement: 深色模式的真实磨砂质感
系统 SHALL 在深色模式下为背景和内容面板提供克制的磨砂材质表达，包括低强度噪点、柔和高光边缘和通透的明暗层次，同时保持文字与图表的清晰可读。

#### Scenario: 深色模式质感增强
- **WHEN** 用户切换到深色模式并浏览主要工作区、卡片和弹窗
- **THEN** 界面呈现更真实的磨砂玻璃层次，视觉更细腻统一，且内容不会因纹理和高光而模糊或失真

## MODIFIED Requirements
### Requirement: 主题样式切换稳定性
系统在进行主题切换时，必须同时保证颜色变量、背景纹理、面板透明度和阴影层级同步更新，且切换后布局尺寸、边框和内容排布不发生异常跳变。

### Requirement: 弹层与面板显示一致性
系统中的 tooltip、modal、详情面板和上传对话区域必须遵循一致的定位与尺寸策略，在不同主题和页面状态下均保持正确显示与可操作性。
