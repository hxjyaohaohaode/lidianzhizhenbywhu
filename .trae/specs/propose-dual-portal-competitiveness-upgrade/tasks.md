# Tasks

- [x] Task 1: 梳理双端目标与信息架构，形成企业端和普通用户端的核心工作台升级方案。
  - [x] SubTask 1.1: 盘点企业端当前高频路径，明确“数据采集-诊断-整改-追踪”四段式闭环中的首屏信息、快捷动作和关键状态。
  - [x] SubTask 1.2: 盘点普通用户端当前高频路径，明确“筛选-对比-论证-结论”四段式闭环中的首屏信息、快捷动作和关键状态。
  - [x] SubTask 1.3: 输出双端首页、分析页和辅助面板的布局优先级与响应式展示要求。

- [x] Task 2: 建立真实数据接口治理与可追溯展示能力。
  - [x] SubTask 2.1: 为内部分析数据、外部行业数据和用户上传数据定义统一的来源元信息结构，包括来源名称、更新时间、可信度、真实/模拟标识和异常状态。
  - [x] SubTask 2.2: 在关键图表、指标卡和分析结论区域增加来源说明、更新时间、异常降级提示和字段映射状态。
  - [x] SubTask 2.3: 制定接口异常时的降级策略与界面反馈，确保不可用数据不会误导用户。

- [x] Task 3: 补齐页面动作闭环，确保所有按钮与快捷操作都有实际功能和反馈。
  - [x] SubTask 3.1: 为企业端和普通用户端建立页面级动作矩阵，列出按钮、入口、浮层动作、快捷键及其目标结果。
  - [x] SubTask 3.2: 为成功、处理中、失败、空态和权限不足态建立统一反馈规范。
  - [x] SubTask 3.3: 对未落地或仅展示型动作补齐实现计划、替代方案或隐藏规则。

- [x] Task 4: 统一界面视觉与图表体验规范，提升双端的高级感和一致性。
  - [x] SubTask 4.1: 定义双端统一设计令牌，包括主题色、卡片层级、间距、圆角、阴影、字体层级和动态效果强度。
  - [x] SubTask 4.2: 为折线图、柱状图、瀑布图、表格卡片和对比图建立统一的样式、单位、空态、来源标识和联动反馈规范。
  - [x] SubTask 4.3: 明确不同分辨率下的布局约束，避免图表溢出、卡片错位和浮层遮挡。

- [x] Task 5: 设计提升市场竞争力的关键增强模块，并限定为真实可行能力。
  - [x] SubTask 5.1: 企业端补充整改任务台、责任归因视图、经营动作跟踪与复盘入口。
  - [x] SubTask 5.2: 普通用户端补充企业对比看板、投资假设卡、证据链摘要和观点留痕。
  - [x] SubTask 5.3: 为每项增强模块标注依赖的数据接口、实现前提和首版范围，避免提出不可落地功能。

- [x] Task 6: 建立发布前体验验收机制，作为版本竞争力门禁。
  - [x] SubTask 6.1: 形成覆盖双端主流程、数据真实性、按钮功能、图表可读性、动效表现和布局稳定性的验收清单。
  - [x] SubTask 6.2: 将验收清单映射到开发自测、产品验收和演示场景检查项。
  - [x] SubTask 6.3: 明确每次版本发布必须通过的关键门槛和回退条件。

# Task Dependencies
- [Task 2] 依赖于 [Task 1] 对双端核心路径与页面结构的梳理结果。
- [Task 3] 依赖于 [Task 1] 明确页面入口与角色高频动作。
- [Task 4] 可与 [Task 2] 并行推进，但图表来源说明需复用 [Task 2.1] 的元信息结构。
- [Task 5] 依赖于 [Task 1] 和 [Task 2]，以确保新增模块既符合角色目标也具备真实数据基础。
- [Task 6] 依赖于 [Task 2]、[Task 3]、[Task 4]、[Task 5] 的输出结果。
