# 图表系统深度优化规格

## Why

当前系统图表布局不协调、重点不突出、交互性差、弹窗体验粗糙，且企业端首页未渲染图表导致两端不对称。用户要求全面升级为高质量可视化图表（雷达图、瀑布图、热力图、三维气泡图、箱型图、散点图、桑基图、折线图），减少文字让图表更突出，深度优化动态交互和弹窗，确保企业端和投资端均展示对应图表。

## What Changes

- **升级所有图表为高质量可视化**：雷达图、瀑布图、热力图、三维气泡图、箱型图、散点图、桑基图、折线图全面重构，提升视觉品质
- **新增桑基图（Sankey Diagram）**：用于展示成本流向、资金流向、风险传导路径
- **大幅精简文字**：移除 widget 副标题、描述、脚注、强调标签、数据源标签等冗余文本，仅保留核心标题
- **企业端首页启用图表渲染**：将已构建但未渲染的 11 个 widget 通过 VisualizationBoard 实际展示
- **深度优化图表交互**：增强 hover 动画、点击反馈、过渡效果、数据点高亮
- **重构弹窗系统**：ChartDetailModal 全面升级为沉浸式详情面板，支持图表放大、数据下钻、关联分析
- **图表自由缩放**：所有图表支持用户随心放大缩小，优化 ChartZoomWrapper 交互体验
- **确保双端图表对称**：企业端和投资端首页/分析页均展示对应图表

## Impact

- Affected code: `src/web/chart-system.tsx`（图表渲染全面重构）、`src/web/chart-renderer.tsx`（对话图表优化）、`src/web/chart-data.ts`（数据配置精简）、`src/web/components/EnterpriseScreen.tsx`（首页启用图表）、`src/web/components/InvestorScreen.tsx`（图表优化）、`src/web/components/ChartZoomWrapper.tsx`（缩放优化）
- Affected systems: 图表渲染系统、交互系统、弹窗系统、页面布局系统
- **BREAKING**: widget 配置中的 subtitle/description/footnote/emphasisTag/dataSources 字段将被移除或忽略

## ADDED Requirements

### Requirement: 高质量图表可视化

系统 SHALL 将所有图表升级为高质量可视化，具体要求：

1. **雷达图**：多边形填充渐变 + 发光边线 + 数据点脉冲动画 + 悬停维度放大 + 基准对比虚线
2. **瀑布图**：浮动柱形渐变填充 + 正负值色彩区分 + 累计连线 + 悬停柱条上浮 + 点击下钻
3. **热力图**：连续色阶渐变 + 单元格圆角 + 悬停光晕扩散 + 点击聚焦 + 色阶图例
4. **三维气泡图**：气泡大小映射 + 颜色渐变 + 悬停放大 + 拖拽交互 + 维度轴标签
5. **箱型图**：中位数线高亮 + 四分位填充 + 离群值标记 + 悬停展开统计详情 + 多组对比
6. **散点图**：数据点渐变 + 悬停光晕 + 趋势线 + 聚类着色 + 点击详情
7. **桑基图**：节点色块 + 流向带宽映射 + 悬停高亮路径 + 点击追踪流向 + 渐变连线
8. **折线图**：渐变面积填充 + 数据点发光 + 阈值线 + 悬停放大 + 区域选择

#### Scenario: 图表视觉品质

- **WHEN** 用户查看任意图表
- **THEN** 图表呈现高质量渐变、发光、动画效果，视觉品质达到专业数据可视化水准

### Requirement: 桑基图（Sankey Diagram）

系统 SHALL 新增桑基图类型，用于展示：

- **企业端**：成本结构流向（原材料→各产品线→毛利）、风险传导路径（外部风险→内部指标→经营结果）
- **投资端**：资金流向（投资→赛道→收益）、风险传导路径（行业风险→企业暴露→投资损失）

桑基图 SHALL 支持：
- 节点色块带标签
- 流向带宽按数值映射
- 悬停高亮完整路径
- 点击节点追踪上下游
- 渐变连线从源色到目标色

#### Scenario: 企业端成本桑基图

- **WHEN** 用户查看企业端首页
- **THEN** 展示成本结构桑基图，显示从原材料成本到各产品线毛利的流向

#### Scenario: 投资端资金桑基图

- **WHEN** 用户查看投资端首页
- **THEN** 展示资金流向桑基图，显示从投资到各赛道的配置流向

### Requirement: 精简文字突出图表

系统 SHALL 大幅精简界面文字，让图表成为视觉焦点：

- **移除** widget 的 subtitle（副标题）、description（描述）、footnote（脚注）、emphasisTag（强调标签）
- **移除** 数据源标签（HeatChip，如"企业采集表单 · 高置信度"）
- **移除** section 副标题和强调文本
- **保留** 仅有意义的标题（≤8字）和必要的轴标签/图例
- **保留** 看板标题但精简为 ≤6字
- **图表容器占比提升**：图表区域占容器面积从约 60% 提升至 85%+

#### Scenario: 界面文字精简

- **WHEN** 用户查看任意页面
- **THEN** 界面文字量减少 60%+，图表视觉占比显著提升，用户一眼聚焦图表内容

### Requirement: 企业端首页启用图表渲染

系统 SHALL 在企业端首页实际渲染图表，使用 VisualizationBoard 组件展示已构建的 11 个 widget：

- 核心指标卡、毛利率走势折线图、利润演变瀑布图、对标对照表
- 斑马纹成本结构表、卡片式分组表格、经营质量雷达图
- 毛利率分布箱型图、毛利率与营收关联散点图、经营三维气泡图、经营质量热力图

替换当前的"经营诊断对话入口"单一卡片为完整图表看板。

#### Scenario: 企业端首页图表展示

- **WHEN** 企业用户进入首页
- **THEN** 页面展示完整的图表看板，包含所有 11 个可视化 widget

### Requirement: 深度优化图表动态交互

系统 SHALL 为所有图表实现极强的动态交互能力：

1. **Hover 动画增强**：
   - 数据点/柱条/扇区悬停时放大 1.15-1.3 倍 + 发光光晕 + 颜色变亮
   - 过渡动画使用 cubic-bezier(0.34, 1.56, 0.64, 1) 弹性曲线
   - 悬停时显示精致浮动信息卡（非简单 tooltip）

2. **点击交互增强**：
   - 点击数据元素触发涟漪扩散动画
   - 点击后其他元素淡出（opacity 0.3），聚焦元素高亮
   - 点击打开沉浸式详情面板

3. **过渡动画**：
   - 图表加载时数据从 0 动画增长至目标值（800ms stagger）
   - 页面切换时图表淡入 + 上浮
   - 数据更新时平滑过渡而非跳变

4. **浮动信息卡**：
   - 替代传统 tooltip，采用毛玻璃卡片设计
   - 显示关键指标 + 同比/环比 + 状态标签
   - 支持迷你趋势线嵌入
   - 出现/消失带缩放 + 淡入淡出动画

#### Scenario: 图表悬停交互

- **WHEN** 用户将鼠标悬停在图表数据点上
- **THEN** 数据点放大并发光，同时显示精致的浮动信息卡，包含关键指标和趋势

#### Scenario: 图表点击聚焦

- **WHEN** 用户点击图表中的数据元素
- **THEN** 触发涟漪动画，其他元素淡出，聚焦元素高亮，并打开沉浸式详情面板

### Requirement: 重构弹窗系统

系统 SHALL 将 ChartDetailModal 全面升级为沉浸式详情面板：

1. **面板设计**：
   - 从中心弹出，背景模糊遮罩
   - 面板宽度 80vw，最大高度 85vh
   - 顶部显示图表标题 + 关闭按钮
   - 主体区域为放大的图表（支持交互）
   - 底部显示关键数据摘要卡片

2. **面板内容**：
   - 放大版原图表（可交互、可缩放）
   - 数据明细表格（可排序、可筛选）
   - 关联指标卡片（同比/环比/排名）
   - 建议追问按钮（跳转对话）

3. **面板动画**：
   - 打开：从点击位置缩放展开 + 背景淡入
   - 关闭：缩小回点击位置 + 背景淡出
   - 过渡时长 300ms

4. **面板交互**：
   - 支持键盘 ESC 关闭
   - 支持点击遮罩关闭
   - 图表在面板内支持缩放和交互

#### Scenario: 图表详情面板

- **WHEN** 用户点击图表数据元素
- **THEN** 从点击位置弹出沉浸式详情面板，显示放大图表、数据明细和关联指标

### Requirement: 图表自由缩放优化

系统 SHALL 优化 ChartZoomWrapper 组件，确保用户可随心放大缩小所有图表：

- 缩放范围扩展至 30%~300%
- 缩放时图表内容自适应重排
- 缩放控件始终可见且不遮挡图表
- 支持触摸屏双指缩放
- 缩放状态持久化（同页面内切换后返回保持缩放）
- 缩放动画流畅（60fps）

#### Scenario: 图表缩放

- **WHEN** 用户在图表上滚动鼠标滚轮或使用缩放控件
- **THEN** 图表平滑缩放，内容自适应，缩放比例显示在控件旁

### Requirement: 双端图表对称展示

系统 SHALL 确保企业端和投资端均展示对应的图表：

**企业端首页**：核心指标卡 + 毛利率走势折线图 + 利润演变瀑布图 + 经营质量雷达图 + 毛利率分布箱型图 + 毛利率营收散点图 + 经营三维气泡图 + 经营质量热力图 + 成本流向桑基图

**投资端首页**：投资指标卡 + 赛道热度柱状图 + 投资画像雷达图 + 行业风险箱型图 + 风险收益散点图 + 投资三维气泡图 + 风险收益热力图 + 资金流向桑基图

**企业端分析页**：DQI/GMPS 诊断面板 + 经营预警表格 + 成本拆解桑基图

**投资端分析页**：DQI/GMPS 诊断面板 + 投资判断对照表 + 资金配置桑基图 + What-If 沙盘

#### Scenario: 企业端首页图表完整

- **WHEN** 企业用户进入首页
- **THEN** 首页展示完整的图表看板，包含折线图、瀑布图、雷达图、箱型图、散点图、气泡图、热力图、桑基图

#### Scenario: 投资端首页图表完整

- **WHEN** 投资用户进入首页
- **THEN** 首页展示完整的图表看板，包含柱状图、雷达图、箱型图、散点图、气泡图、热力图、桑基图

## MODIFIED Requirements

### Requirement: 图表缩放功能

原缩放范围 50%~200% 扩展为 30%~300%，增加触摸屏支持和缩放状态持久化。

### Requirement: 图表交互

原 tooltip 和点击弹窗升级为浮动信息卡 + 沉浸式详情面板，交互体验全面增强。

### Requirement: widget 配置

widget 配置中的 subtitle、description、footnote、emphasisTag、dataSources 字段标记为废弃，渲染时不再使用。

## REMOVED Requirements

### Requirement: 冗余文字展示

**Reason**: 用户要求减少界面文字，让图表更突出
**Migration**: 移除 widget 副标题、描述、脚注、强调标签、数据源标签的渲染逻辑
