# 系统设计方案 - 数学模型集成补充

本文档是对《系统设计方案.md》的补充，详细说明如何将两个核心数学模型（DQI模型和GMPS模型）深度集成到智能诊断系统中。

---

## 一、数学模型在系统中的定位

### 1.1 核心地位

两个数学模型是智能诊断系统的**算法底座**，为多智能体协作提供量化分析能力：

```
┌─────────────────────────────────────────────────────────┐
│              智能诊断系统架构                              │
├─────────────────────────────────────────────────────────┤
│  用户交互层：企业端门户 + 投资端门户                        │
├─────────────────────────────────────────────────────────┤
│  API网关层：请求路由、认证授权、流量控制                    │
├─────────────────────────────────────────────────────────┤
│  多智能体协作层：                                          │
│    ├─ 任务编排智能体（调度中心）                            │
│    ├─ 数据收集智能体 → 调用数学模型接口                     │
│    ├─ 数学分析智能体 → 执行数学模型计算 ★核心★             │
│    ├─ 行业检索智能体 → 提供外部数据输入                     │
│    ├─ 证据审查智能体 → 验证模型计算结果                     │
│    └─ 表达生成智能体 → 生成诊断报告                        │
├─────────────────────────────────────────────────────────┤
│  算法引擎层：                                              │
│    ├─ DQI模型引擎（经营质量评价）★核心算法★                │
│    ├─ GMPS模型引擎（毛利承压分析）★核心算法★               │
│    └─ RAG检索引擎（实时信息获取）                          │
├─────────────────────────────────────────────────────────┤
│  数据层：企业财务数据 + 行业指数 + 原材料价格 + 向量数据库    │
└─────────────────────────────────────────────────────────┘
```

### 1.2 模型作用范围

| 模型 | 作用范围 | 核心功能 | 输出结果 |
|------|----------|----------|----------|
| **DQI模型** | 经营质量评价 | 动态评估企业经营质量变化趋势 | DQI指数、驱动因素、趋势判断 |
| **GMPS模型** | 毛利承压分析 | 量化分析毛利承压程度并预测风险 | GMPS指数、承压等级、风险概率 |

## 二、数学模型与智能体的协作流程

### 2.1 完整诊断流程

```
用户发起诊断请求
    ↓
[任务编排智能体]
    ├─ 解析用户意图（毛利分析/经营质量评价/综合诊断）
    ├─ 确定需要的数学模型（DQI/GMPS/两者）
    └─ 分配任务给其他智能体
    ↓
[数据收集智能体]
    ├─ 收集企业财务数据（净利润、营业收入、现金流等）
    ├─ 收集经营数据（销量、产量、库存等）
    ├─ 调用行业检索智能体获取外部数据
    │   ├─ 碳酸锂价格（用于GMPS的B层指标）
    │   └─ 行业指数波动率（用于GMPS的D层指标）
    └─ 验证数据完整性和合理性
    ↓
[数学分析智能体] ★核心计算环节★
    ├─ 根据任务类型选择模型
    │   ├─ DQI模型计算
    │   │   ├─ 计算ROE = 净利润 / 平均净资产 × 100%
    │   │   ├─ 计算Growth = (当期营收 - 上期营收) / 上期营收
    │   │   ├─ 计算OCF比率 = 经营现金流 / 营业收入
    │   │   ├─ 计算DQI = w1·(ROE_t/ROE_{t-1}) + w2·(Growth_t/Growth_{t-1}) + w3·(OCF_t/OCF_{t-1})
    │   │   ├─ 识别驱动因素 = argmax{ROE比率, Growth比率, OCF比率}
    │   │   └─ 判断趋势：DQI>1改善，DQI=1稳定，DQI<1恶化
    │   │
    │   └─ GMPS模型计算
    │       ├─ 计算A层指标（毛利率同比、营收成本增速差）
    │       ├─ 计算B层指标（碳酸锂价格同比、单位成本同比）
    │       ├─ 计算C层指标（库存同比、产销率、制造费用占比）
    │       ├─ 计算D层指标（行业指数波动率）
    │       ├─ 计算E层指标（现金流比率、资产负债率）
    │       ├─ 标准化打分（根据阈值映射到0-100分）
    │       ├─ 加权求和：GMPS = Σ w_k · score_k
    │       ├─ 等级划分：GMPS<40低压，40-70中压，≥70高压
    │       └─ Logistic预测：P = 1/(1+e^{-(β_0+β_1·GMPS+...)})
    │
    └─ 返回计算结果
    ↓
[证据审查智能体]
    ├─ 验证计算结果合理性
    │   ├─ DQI范围检查（通常0.5-2.0）
    │   ├─ GMPS范围检查（0-100）
    │   └─ 概率范围检查（0-1）
    ├─ 交叉验证数据一致性
    │   ├─ 检查财务数据逻辑关系
    │   └─ 检查指标计算正确性
    └─ 评估结果可信度
    ↓
[表达生成智能体]
    ├─ 基于模型结果生成诊断报告
    │   ├─ 经营质量评价报告（基于DQI）
    │   ├─ 毛利承压分析报告（基于GMPS）
    │   └─ 综合诊断报告（整合两者）
    ├─ 生成可视化建议
    │   ├─ DQI趋势图
    │   ├─ GMPS雷达图
    │   └─ 驱动因素分解图
    └─ 生成行动建议
        ├─ 基于驱动因素的建议
        └─ 基于风险等级的建议
    ↓
返回用户
```

### 4.1 DQI模型API

```typescript
// POST /api/v1/diagnosis/dqi/calculate
router.post('/dqi/calculate', async (req, res) => {
  const { enterpriseId, currentPeriod, baselinePeriod } = req.body;
  
  // 1. 获取财务数据
  const currentData = await getFinancialData(enterpriseId, currentPeriod);
  const baselineData = await getFinancialData(enterpriseId, baselinePeriod);
  
  // 2. 计算ROE
  const currentROE = calculateROE(
    currentData.netProfit,
    currentData.beginningEquity,
    currentData.endingEquity
  );
  const baselineROE = calculateROE(
    baselineData.netProfit,
    baselineData.beginningEquity,
    baselineData.endingEquity
  );
  
  // 3. 计算Growth
  const currentGrowth = (currentData.revenue - baselineData.revenue) / baselineData.revenue;
  const baselineGrowth = await getPreviousGrowth(enterpriseId, baselinePeriod);
  
  // 4. 计算OCF比率
  const currentOCFRatio = currentData.operatingCashFlow / currentData.revenue;
  const baselineOCFRatio = baselineData.operatingCashFlow / baselineData.revenue;
  
  // 5. 计算DQI
  const weights = { w1: 0.4, w2: 0.3, w3: 0.3 };
  const roeRatio = currentROE / baselineROE;
  const growthRatio = currentGrowth / baselineGrowth;
  const ocfRatio = currentOCFRatio / baselineOCFRatio;
  
  const dqi = weights.w1 * roeRatio + weights.w2 * growthRatio + weights.w3 * ocfRatio;
  
  // 6. 识别驱动因素
  const driver = identifyDriver(roeRatio, growthRatio, ocfRatio);
  
  // 7. 判断状态
  const status = interpretDQI(dqi);
  
  // 8. 保存结果
  await saveDQIResult({
    enterpriseId,
    periodDate: currentPeriod,
    dqi,
    status,
    driver,
    roe: currentROE,
    growth: currentGrowth,
    ocfRatio: currentOCFRatio,
    roeRatio,
    growthRatio,
    ocfRatioChange: ocfRatio,
    profitabilityContribution: weights.w1 * roeRatio,
    growthContribution: weights.w2 * growthRatio,
    cashflowContribution: weights.w3 * ocfRatio
  });
  
  res.json({
    success: true,
    data: {
      dqi,
      status,
      driver,
      decomposition: {
        profitabilityContribution: weights.w1 * roeRatio,
        growthContribution: weights.w2 * growthRatio,
        cashflowContribution: weights.w3 * ocfRatio
      }
    }
  });
});
```

### 4.2 GMPS模型API

```typescript
// POST /api/v1/diagnosis/gmps/calculate
router.post('/gmps/calculate', async (req, res) => {
  const { enterpriseId, currentPeriod, baselinePeriod } = req.body;
  
  // 1. 获取企业数据
  const currentData = await getFinancialData(enterpriseId, currentPeriod);
  const baselineData = await getFinancialData(enterpriseId, baselinePeriod);
  
  // 2. 获取外部数据
  const currentLithiumPrice = await getLithiumPrice(currentPeriod);
  const baselineLithiumPrice = await getLithiumPrice(baselinePeriod);
  const industryVolatility = await getIndustryVolatility(currentPeriod);
  
  // 3. 计算A层指标
  const gpmYoy = (currentData.grossMargin - baselineData.grossMargin) / baselineData.grossMargin;
  const revenueGrowth = (currentData.revenue - baselineData.revenue) / baselineData.revenue;
  const costGrowth = (currentData.operatingCost - baselineData.operatingCost) / baselineData.operatingCost;
  const revCostGap = revenueGrowth - costGrowth;
  
  // 4. 计算B层指标
  const liPriceYoy = (currentLithiumPrice - baselineLithiumPrice) / baselineLithiumPrice;
  const currentUnitCost = currentData.operatingCost / currentData.salesVolume;
  const baselineUnitCost = baselineData.operatingCost / baselineData.salesVolume;
  const unitCostYoy = (currentUnitCost - baselineUnitCost) / baselineUnitCost;
  
  // 5. 计算C层指标
  const invYoy = (currentData.inventory - baselineData.inventory) / baselineData.inventory;
  const saleProdRatio = currentData.salesVolume / currentData.productionVolume;
  const mfgCostRatio = currentData.manufacturingExpense / currentData.operatingCost;
  
  // 6. 计算D层指标
  const indVol = industryVolatility;
  
  // 7. 计算E层指标
  const cfoRatio = currentData.operatingCashFlow / currentData.revenue;
  const lev = currentData.totalLiabilities / currentData.totalAssets;
  
  // 8. 标准化打分
  const features = {
    gpmYoy,
    revCostGap,
    liPriceYoy,
    unitCostYoy,
    invYoy,
    saleProdRatio,
    mfgCostRatio,
    indVol,
    cfoRatio,
    lev
  };
  
  const scores = calculateScores(features);
  
  // 9. 计算GMPS
  const weights = {
    gpmYoy: 0.14,
    revCostGap: 0.11,
    liPriceYoy: 0.10,
    unitCostYoy: 0.12,
    invYoy: 0.09,
    saleProdRatio: 0.10,
    mfgCostRatio: 0.12,
    indVol: 0.07,
    cfoRatio: 0.08,
    lev: 0.07
  };
  
  let gmps = 0;
  for (const [key, score] of Object.entries(scores)) {
    gmps += score * weights[key];
  }
  
  // 10. 等级划分
  const level = gmps < 40 ? '低压' : gmps < 70 ? '中压' : '高压';
  
  // 11. 计算维度得分
  const dimensionScores = calculateDimensionScores(scores, weights);
  
  // 12. Logistic预测
  const betaParams = {
    beta_0: -2.5,
    beta_1: 0.05,
    beta_2: 0.02,
    beta_3: 0.03,
    beta_4: 0.02,
    beta_5: 0.01,
    beta_6: 0.02
  };
  
  const probability = predictProbability(gmps, dimensionScores, betaParams);
  const riskLevel = probability < 0.33 ? '低风险' : probability < 0.66 ? '中风险' : '高风险';
  
  // 13. 保存结果
  await saveGMPSResult({
    enterpriseId,
    periodDate: currentPeriod,
    gmps,
    level,
    probabilityNextQuarter: probability,
    riskLevel,
    dimensionScores,
    featureScores: scores
  });
  
  res.json({
    success: true,
    data: {
      gmps,
      level,
      probabilityNextQuarter: probability,
      riskLevel,
      dimensionScores,
      featureScores: scores
    }
  });
});
```

### 8.3 关键公式确认

所有公式均已正确理解并实现：

✅ **DQI模型**：
- ROE = 净利润 / 平均净资产 × 100%
- DQI_t = w1·(ROE_t/ROE_{t-1}) + w2·(Growth_t/Growth_{t-1}) + w3·(OCF_t/OCF_{t-1})
- 权重：w1=0.4, w2=0.3, w3=0.3

✅ **GMPS模型**：
- 10个特征变量的计算公式
- 标准化打分函数（越大越危险/越小越危险）
- GMPS = Σ w_k · score_k
- Logistic回归：P = 1/(1+e^{-(β_0+β_1·GMPS+...)})

所有公式实现均严格遵循原始数学建模文档的定义！
