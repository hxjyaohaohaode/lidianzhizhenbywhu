## 1. 项目概述与定位

### 1.1 项目背景

当前锂电池行业正经历前所未有的复杂市场环境：

**碳酸锂价格剧烈波动**（基于现有数据）：
- 2026 年 1 月初：138,000 元/吨
- 2026 年 1 月下旬峰值：175,250 元/吨（涨幅 27%）
- 2026 年 2 月初低谷：124,100 元/吨（跌幅 29%）
- 2026 年 4 月初：158,000 元/吨
- **波动幅度超过 40%**，企业成本管控压力巨大

**资本市场剧烈震荡**（基于现有数据）：
- 创业板指数从 2024 年 9 月的 1,520 点飙升至 2026 年 3 月的 3,426 点（涨幅 125%）
- 中证动力电池指数近期在 3,800-4,150 点区间剧烈波动
- **行业估值与基本面背离风险**加剧

**核心痛点**：
1. 锂电池企业缺乏有效的毛利承压预警和诊断工具
2. 投资者难以获取可信的企业经营质量评估信息
3. 现有分析工具缺乏实时性和可解释性
4. 行业信息分散，缺乏权威的整合分析平台

### 1.2 项目定位

**成为锂电池行业最具专业性和可信度的智能诊断平台**

核心价值主张：
- **数学模型驱动**：基于专业毛利承压模型和经营质量动态评价模型
- **多智能体协作**：8 个专业化智能体 +3 个大模型协同工作
- **实时信息获取**：RAG 技术确保信息时效性和可信度
- **双端个性化服务**：企业端和投资端差异化体验

### 1.3 目标用户群体

## 3. 多智能体协作框架

### 3.1 智能体职责定义

| 智能体 ID | 名称 | 核心职责 | 输入 | 输出 | 协作对象 |
|----------|------|---------|------|------|---------|
| `taskOrchestrator` | 任务编排智能体 | 分析用户需求，制定诊断计划，协调各智能体工作 | 用户查询、角色信息、历史对话 | 任务分解结果、执行计划、最终汇总 | 所有智能体 |
| `memoryManagement` | 记忆管理智能体 | 管理用户记忆数据，提供历史上下文，维护 3D 记忆树 | 用户 ID、会话 ID、新数据 | 相关记忆片段、记忆树结构 | 所有智能体 |
| `dataGathering` | 数据获取智能体 | 采集企业内部数据、财务数据、运营数据 | 数据需求清单、企业标识 | 结构化数据集、数据质量报告 | mathAnalysis, dataUnderstanding |
| `dataUnderstanding` | 数据理解智能体 | 解析数据含义，识别异常，提取关键信息 | 原始数据、数据字典 | 数据洞察、异常标记、关键指标 | taskOrchestrator, mathAnalysis |
| `mathAnalysis` | 数学分析智能体 | 执行数学模型计算，进行毛利承压分析、经营质量评估 | 财务数据、模型参数 | 计算结果、风险评级、趋势预测 | 所有智能体 |
| `industryRetrieval` | 行业检索智能体 | 检索行业信息、政策动态、竞争对手情报 | 检索关键词、行业分类 | 行业报告、政策信息、竞争情报 | evidenceReview, dataUnderstanding |
| `evidenceReview` | 证据审查智能体 | 验证信息可信度，多源信息比对，时效性检查 | 原始信息、来源列表 | 可信度评分、信息筛选结果、引用列表 | 所有智能体 |
| `expressionGeneration` | 表达生成智能体 | 生成分析报告、优化语言表达、制作可视化建议 | 分析结果、用户角色、报告模板 | 结构化报告、可视化配置、行动建议 | 所有智能体 |

#### 5.2.2 数学模型模块

**毛利承压模型**（基于现有 Schema）：

输入指标（12 项）：
- 当前毛利率、基期毛利率
- 当前收入、基期收入
- 当前成本、基期成本
- 当前销量、基期销量
- 当前库存费用、基期库存费用
- 当前制造费用、基期制造费用
- 当前营业成本、基期营业成本

计算逻辑：
```javascript
// 毛利压力指数计算
function calculateGrossMarginPressure(input) {
  // 1. 毛利率变化率
  const marginChange = input.currentGrossMargin - input.baselineGrossMargin;
  
  // 2. 成本压力指数
  const costPressure = (input.currentCost / input.currentRevenue) - 
                       (input.baselineCost / input.baselineRevenue);
  
  // 3. 销量影响指数
  const volumeImpact = (input.currentSalesVolume - input.baselineSalesVolume) / 
                       input.baselineSalesVolume;
  
  // 4. 库存压力指数
  const inventoryPressure = input.currentInventoryExpense / input.currentRevenue;
  
  // 5. 综合压力指数（加权）
  const pressureIndex = 
    -marginChange * 40 +  // 毛利率下降为正贡献
    costPressure * 30 +   // 成本上升为正贡献
    -volumeImpact * 20 +  // 销量下降为正贡献
    inventoryPressure * 10; // 库存费用高为正贡献
