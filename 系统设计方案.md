## 第一章 项目概述

### 1.1 项目背景

锂电池行业作为新能源产业链的核心环节，正面临原材料价格波动、市场竞争加剧、技术迭代加速等多重挑战。碳酸锂价格的大幅波动直接影响企业毛利水平，而经营质量的变化则关系到企业的长期可持续发展。

本系统基于已建立的**毛利承压分析数学模型**和**经营质量变化动态评价模型**，采用多智能体协作架构，结合 RAG（检索增强生成）技术，为锂电池企业提供实时、准确、可解释的智能诊断服务。

### 1.2 项目定位

**系统定位**：面向锂电池企业的智能化诊断分析平台

**目标用户**：
- **企业端**：锂电池制造企业经营管理者、财务分析师
- **投资端**：机构投资者、行业分析师、投资顾问

**核心价值**：
1. 基于数学模型的量化分析，确保诊断结果科学可靠
2. 多智能体协作，实现专业化分工和高效协同
3. RAG 技术实时获取最新行业信息，保证诊断时效性
4. 双门户设计，满足企业和投资者不同视角需求

### 1.3 技术基础

**现有项目基础**：
- 前端：React + Vite + TypeScript
- 后端：Express + Node.js
- 数据验证：Zod Schema
- 多智能体架构：8 个专业化智能体
- 大模型支持：DeepSeek-Reasoner、GLM-5、Qwen3.5-Plus

**数学模型基础**：
1. **毛利承压分析模型**：量化分析原材料价格波动对毛利的影响
2. **经营质量动态评价模型**：多维度评估企业经营质量变化趋势

**数据基础**：
- 碳酸锂价格历史数据（2026 年 1-4 月）
- 行业波动指数：创业板指数（2011-2026 年）
- 行业波动指数：中证动力电池指数

### 2.2 多智能体协作架构

基于现有代码 [`agents.js`](file:///c:/Users/LENOVO/Desktop/面向锂电池企业毛利承压与经营质量变化的智能诊断系统/dist/server/shared/agents.js#L11-L18)，系统包含 8 个核心智能体：

#### 2.2.1 智能体角色定义

**1. 任务编排智能体 (Task Orchestrator)**
- **使用模型**：DeepSeek-Reasoner（强推理能力）
- **职责**：整体任务分解和智能体调度
- **功能**：解析用户查询意图、分解复杂任务、调度其他智能体、汇总结果

**2. 记忆管理智能体 (Memory Management)**
- **使用模型**：GLM-5（高效文本处理）
- **职责**：用户会话记忆和历史上下文管理
- **功能**：存储用户历史对话、提取关键记忆点、管理会话状态

**3. 数据收集智能体 (Data Gathering)**
- **使用模型**：GLM-5
- **职责**：企业内部数据采集和验证
- **功能**：收集企业财务数据、验证数据完整性、计算基础指标

**4. 数据理解智能体 (Data Understanding)**
- **使用模型**：Qwen3.5-Plus（强大数据分析能力）
- **职责**：数据解读和初步分析
- **功能**：解读财务数据含义、识别数据趋势、发现异常波动

**5. 数学分析智能体 (Math Analysis)**
- **使用模型**：DeepSeek-Reasoner（强数学推理）
- **职责**：基于数学模型的量化分析
- **功能**：执行毛利承压分析模型、执行经营质量评价模型、计算关键指标

**6. 行业检索智能体 (Industry Retrieval)**
- **使用模型**：Qwen3.5-Plus + RAG
- **职责**：行业信息和市场数据检索
- **功能**：检索最新行业动态、获取原材料价格信息、收集政策法规

**7. 证据审查智能体 (Evidence Review)**
- **使用模型**：DeepSeek-Reasoner（强批判性思维）
- **职责**：信息可信度评估和交叉验证
- **功能**：评估信息来源可信度、交叉验证数据一致性、识别信息时效性

**8. 表达生成智能体 (Expression Generation)**
- **使用模型**：Qwen3.5-Plus（强文本生成）
- **职责**：最终报告生成和可视化建议
- **功能**：生成诊断报告、提供可视化建议、生成行动建议

#### 2.2.2 智能体协作流程

```
用户查询
   ↓
[任务编排智能体] - 解析意图、分解任务
   ↓
[并行执行]
  ├─ [数据收集] → [数据理解] (企业内部数据分析)
  ├─ [行业检索] (RAG 技术支持)
  └─ [记忆管理] (提供历史上下文)
   ↓
[数学分析智能体] - 模型计算
   ↓
[证据审查智能体] - 可信度评估
   ↓
[表达生成智能体] - 生成报告
   ↓
返回用户
```

### 4.2 数据模型设计

#### 4.2.1 核心数据表

**1. 企业基本信息表 (enterprises)**
```sql
CREATE TABLE enterprises (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(200) NOT NULL,
    code VARCHAR(50) UNIQUE,  -- 股票代码或企业编码
    region VARCHAR(100),      -- 所属地区
    scale VARCHAR(20),        -- large/medium/small
    industry_segment VARCHAR(100), -- 正极/负极/电解液/隔膜
    listed_status BOOLEAN DEFAULT false,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**2. 财务数据表 (financial_data)**
```sql
CREATE TABLE financial_data (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    enterprise_id UUID REFERENCES enterprises(id),
    period VARCHAR(20) NOT NULL,  -- 2026-Q1
    period_type VARCHAR(10),      -- quarterly/annual
    
    -- 毛利相关
    gross_margin DECIMAL(5,2),
    revenue DECIMAL(15,2),
    cost DECIMAL(15,2),
    sales_volume DECIMAL(15,2),
    
    -- 经营质量相关
    production_volume DECIMAL(15,2),
    manufacturing_expense DECIMAL(15,2),
    operating_cost DECIMAL(15,2),
    operating_cash_flow DECIMAL(15,2),
    total_assets DECIMAL(15,2),
    total_liabilities DECIMAL(15,2),
    
    -- 辅助指标
    inventory_expense DECIMAL(15,2),
    rd_expense DECIMAL(15,2),
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(enterprise_id, period)
);
```

**3. 诊断报告表 (diagnostic_reports)**
```sql
CREATE TABLE diagnostic_reports (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    enterprise_id UUID REFERENCES enterprises(id),
    user_id UUID REFERENCES users(id),
    report_type VARCHAR(50),      -- full/quick
    role_type VARCHAR(20),        -- enterprise/investor
    generated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- 核心指标
    gross_margin_pressure_index DECIMAL(5,2),
    operating_quality_score DECIMAL(5,2),
```

```javascript
// realtime-rag.js 核心实现
async function retrieveRelevantDocuments(query, options = {}) {
    const { topK = 10, minCredibility = 0.7, dateRange = '3months' } = options;
    
    // 1. 生成查询向量
    const queryEmbedding = await generateEmbedding(query);
    
    // 2. 向量相似度检索
    const vectorResults = await pg.query(`
        SELECT id, title, content, source_url, publish_date,
               credibility_score,
               1 - (embedding <=> $1::vector) AS similarity
        FROM industry_knowledge_base
        WHERE credibility_score >= $2
          AND publish_date >= $3
        ORDER BY embedding <=> $1::vector
        LIMIT $4
    `, [queryEmbedding, minCredibility, getDateRange(dateRange), topK * 2]);
    
    // 3. 重排序 (考虑时效性和相关性)
    const rerankedResults = rerankResults(vectorResults, {
        recencyWeight: 0.3,
        credibilityWeight: 0.4,
        relevanceWeight: 0.3
    });
    
    // 4. 返回 Top-K 结果
    return rerankedResults.slice(0, topK);
}

// 可信度评估
async function evaluateCredibility(document) {
    const factors = {
        sourceAuthority: rateSourceAuthority(document.source_url),
        recency: calculateRecencyScore(document.publish_date),
        citationCount: normalizeCitationCount(document.citation_count),
        consistency: await checkConsistencyWithOtherSources(document.content)
    };
    
    const weights = {
        sourceAuthority: 0.35,
        recency: 0.25,
        citationCount: 0.15,
        consistency: 0.25
    };
    
    const overallCredibility = Object.keys(factors).reduce(
        (sum, key) => sum + factors[key] * weights[key], 0
    );
    
    return {
        overall: overallCredibility,
        factors,
        level: overallCredibility >= 0.8 ? 'high' 
               : overallCredibility >= 0.6 ? 'medium' : 'low'
    };
}
```

```javascript
async function handleDiagnosticRequest(request) {
    // 1. 请求验证
    const validatedInput = validateRequest(request);
    
    // 2. 创建会话
    const session = await createSession(validatedInput);
    
    // 3. 任务编排
    const taskPlan = await taskOrchestrator.analyze(validatedInput);
    
    // 4. 并行数据收集
    const [internalData, externalData, memoryContext] = await Promise.all([
        dataGathering.collect(validatedInput.enterpriseName),
        industryRetrieval.retrieve(taskPlan.searchQueries),
        memoryManagement.getContext(session.userId)
    ]);
    
    // 5. 数据理解

    // 6. 数学模型计算
    const mathResults = await mathAnalysis.calculate({
        grossMarginInput: validatedInput.grossMarginInput,
        operatingQualityInput: validatedInput.operatingQualityInput,
        externalContext: externalData
    });
    
    // 7. 证据审查
    const evidenceReview = await evidenceReview.verify({
        data: internalData,
        externalInfo: externalData,
        mathResults
    });
    
    // 8. 表达生成
    const finalReport = await expressionGeneration.generate({
        role: validatedInput.role,
        insights,
        mathResults,
        evidenceReview,
        memoryContext
    });
    
    // 9. 存储结果
    await saveReport(session.id, finalReport);
    
    // 10. 返回结果
    return formatResponse(finalReport);
}
```
